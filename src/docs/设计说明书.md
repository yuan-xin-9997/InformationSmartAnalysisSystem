# 设计说明书 - 信息智能分析系统

## 1. 总体架构

```
[浏览器(Vue3 SPA)] ──HTTP──► [FastAPI 后端]
                                ├─ 认证/权限 (JWT + password.txt)
                                ├─ 信息源适配器 (website / local_folder / freshrss)
                                │     └─► WebFetch 服务 (网页抓取)
                                ├─ 分析引擎 ─► LLM (OpenAI 兼容)
                                ├─ 后台 Worker (ThreadPoolExecutor)
                                └─ SQLite (SQLAlchemy ORM)
```

前后端分离、单服务部署：FastAPI 同时提供 REST API 与前端静态资源托管（SPA 兜底）。

## 2. 技术选型

|---|---|

## 4. 数据库设计（SQLite）

| 表 | 关键字段 | 说明 |
|---|---|---|
| `users` | id, username, password_hash, password_fingerprint, role | 从 password.txt 同步；fingerprint 用于跳过重复哈希 |
| `page_permissions` | user_id, page_key | 普通用户可见页面 |
| `info_sources` | id, name, type, config(JSON), status, last_sync_at, last_error, item_count | 信息源 |
| `info_items` | id, source_id, external_id, title, url, content, content_hash, published_at, analyzed | 抓取条目；(source_id, external_id) 唯一 |
| `analysis_tasks` | id, name, description, config(JSON) | 分析任务 |
| `task_sources` | task_id, source_id, last_analyzed_item_id, last_analyzed_at | **增量水位线** |
| `task_runs` | id, kind(analysis/sync), ref_id, ref_name, mode, status, started_at, finished_at, summary, error | 一次执行 |
| `analysis_results` | task_run_id, task_id, source_id, info_item_id, result_type, content | 分析产出 |
| `task_logs` | run_id, level, message, created_at | 运行日志 |

时间字段统一存 UTC，接口/前端展示转北京时间。

## 5. 模块设计

### 5.1 认证与权限（core/security.py, core/deps.py）

- 启动与登录时解析 `password.txt`，bcrypt 哈希后 upsert 到 `users`（用 `password_fingerprint` 跳过未变更项）。
- 登录返回 JWT（含 username/role）；`get_current_user` 解析 JWT；`require_page(page_key)` 校验页面权限（admin 全放行）。
- 页面权限键：dashboard / info_sources / analysis_tasks / analysis_result / task_center / system_config / permission（permission 仅 admin）。

### 5.2 信息源适配器（services/info_source/）

统一抽象接口：
```
InfoSourceAdapter.check_status() -> SourceStatus
InfoSourceAdapter.fetch_new_items(since=None, known_ids=None) -> list[InfoItemData]
```
`fetch_new_items` 采用「增量 + 回补」：返回 mtime 晚于 `since` 或不在 `known_ids` 中的条目，跳过已索引且未变更的条目（不重读内容），从而首次同步入库全部、后续同步只处理新增/变更。

- `website.py`：调用 WebFetch 服务 `POST /v1/fetch` 获取 HTML，BeautifulSoup 解析文章链接与正文；`known_ids`（已索引 URL）跳过，只抓新文章。
- `local_folder.py`：扫描目录，按扩展名抽取文本（pdf→PyMuPDF，docx→python-docx，其他→原生/BS4）。
- `freshrss.py`：Google Reader API（ClientLogin 认证 → stream/contents 拉取条目）。
- `factory.py`：按类型实例化适配器并校验必填配置。
- `sync.py`：同步任务，传入 `since=source.last_sync_at` 与 `known_ids`（批量加载已存在 external_id），按 (source_id, external_id) upsert，content_hash 检测内容变更；批量去重避免 N+1 查询。

### 5.3 分析引擎（services/analysis/）

- `llm_client.py`：OpenAI 兼容 `POST {base_url}/chat/completions`，Bearer 鉴权；初始化校验 api_key（占位符 / 非 ASCII / 未配置）给出清晰中文错误，请求超时单独提示「endpoint 不可达或响应过慢」。
- `prompts.py`：默认逐条/汇总提示词模板（可在任务 config 中覆盖）。
- `engine.py`：`run_analysis(run_id, task_id, mode)` 编排：
  1. 遍历任务绑定的源；
  2. 增量：取 `id > 水位线` 的条目；全量：取全部；
  3. per_item 逐条调 LLM 存结果；aggregate 拼接送 LLM 存汇总结果；
  4. 推进 `task_sources` 水位线；
  5. 全程写 `task_logs`，更新 `task_runs` 状态/摘要。

### 5.4 后台 Worker（services/worker.py）

进程级 `ThreadPoolExecutor`。API 创建 `task_runs(pending)` 后提交作业到 worker；作业在新线程用独立 DB 会话执行并更新状态/日志。前端轮询任务中心查看进度。

## 6. API 设计（均位于 `/api` 前缀）

| 模块 | 端点 |
|---|---|
| 认证 | `POST /api/auth/login`、`GET /api/auth/me`、`POST /api/auth/logout` |
| 权限管理 | `GET /api/users`、`GET /api/users/pages`、`GET/PUT /api/users/{id}/permissions` |
| 系统配置 | `GET /api/config` |
| 任务中心 | `GET /api/task-center/runs`、`GET /api/task-center/runs/{id}`、`GET .../logs`、`DELETE .../runs/{id}` |
| 信息源 | `GET /api/info-sources/types`、`GET/POST/PUT/DELETE /api/info-sources`、`POST .../{id}/check`、`POST .../{id}/sync`、`GET .../{id}/status`、`GET .../{id}/items` |
| 分析任务 | `GET/POST/PUT/DELETE /api/analysis-tasks`、`POST .../{id}/run`、`GET .../{id}/sources`、`GET .../{id}/results`、`GET /api/analysis-results` |

## 7. 关键流程：增量分析

```
POST /api/analysis-tasks/{id}/run {mode:"incremental"}
  └─ 创建 task_runs(pending) → 提交 worker
       └─ engine.run_analysis
            for source in task.sources:
                items = SELECT * FROM info_items WHERE source_id=? AND id > 水位线
                per_item: LLM 分析每条 → analysis_results
                aggregate: LLM 汇总本批 → analysis_results
                推进 task_sources.last_analyzed_item_id
            task_runs → succeeded + 摘要
       └─ 全程 task_logs + 文件日志
前端轮询任务中心 / 分析结果页查看
```

## 8. 部署与运维

见 `README.md`「部署方式」「运维方式」。支持 ps1/sh 脚本与 Linux systemd；Jenkins 流水线见 `JenkinsConfig/Jenkinsfile`。

### 配置与密钥管理

- 主配置 `config/app.json`：随代码部署，敏感字段为占位符。
- 本地覆盖 `config/env.local`：存放真实敏感配置（LLM Key、JWT 密钥、WebFetch Key 等），以 `ISAS_*` 环境变量覆盖 `app.json`；**不被 Jenkins 部署覆盖**（Jenkinsfile rsync 排除 `config/env.local`）、**不入 Git**（.gitignore 忽略）；`start.sh`/`run.sh` 启动时自动 `source`，修改后重启生效；建议 `chmod 600`。模板见 `config/env.local.example`。
- 配置优先级：环境变量（含 `env.local`）> `app.json`。
- 启停脚本（`start.sh`/`stop.sh`/`status.sh`）在以非部署属主用户运行时，自动 `sudo` 到部署目录属主（`stat` 探测）再执行，避免 `logs/`/`data/` 写权限问题。
- 端口以 `config/app.json` 的 `server.port` 为唯一来源，各脚本/Jenkinsfile/systemd 均从中读取（`ISAS_SERVER_PORT` 可覆盖）。
