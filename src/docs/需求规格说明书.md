# 需求规格说明书 — 信息智能分析系统

## 1. 系统概述

信息智能分析系统（InformationSmartAnalysisSystem）面向给定信息源进行智能分析。用户可添加多种信息源、创建分析任务并绑定信息源、查看信息源状态、触发分析（含基于新增内容的增量分析），分析过程调用大模型接口。

## 2. 用户角色

| 角色 | 说明 |
|---|---|
| 管理员（admin） | 拥有全部页面权限；可管理用户页面权限；可操作所有功能 |
| 普通用户（user） | 可见页面由管理员在权限管理页配置 |

用户与角色维护在 `data/password.txt`（格式 `username:password:role`），修改后新用户在下次登录时自动同步到数据库。

## 3. 功能需求

### 3.1 登录与权限（基础模块）

- FR-AUTH-1：系统支持用户名/密码登录，凭证来源于 `data/password.txt`。
- FR-AUTH-2：登录成功颁发 JWT；未认证请求被拒绝（401）。
- FR-AUTH-3：管理员默认拥有全部页面权限；普通用户的可见页面由管理员配置。
- FR-AUTH-4：前端按权限渲染侧边栏，后端按页面权限拦截未授权 API（403）。

### 3.2 信息源管理（核心）

- FR-SRC-1：支持添加信息源，类型包括：官方网站、指定本地文件夹、FreshRSS 指定源。
- FR-SRC-2：信息源支持编辑、删除。
- FR-SRC-3：可对信息源进行健康检查与手动同步；同步抓取新内容并去重存储。同步采用「增量 + 回补」：跳过已索引且未变更的条目，只读取新增/变更/尚未入库的条目，故大目录首次同步入库全部、后续同步仅处理增量。`max_items` 控制单次同步上限（本地文件夹默认 100000），超出部分下次同步继续回补。
- FR-SRC-4：展示信息源状态（ok/warning/error）、最近同步时间、条目数、最近错误。
- FR-SRC-5：可查看某信息源下的信息条目列表与详情。

### 3.3 分析任务（核心）

- FR-TASK-1：可创建分析任务，并绑定一个或多个信息源作为信息来源范围。
- FR-TASK-2：每个任务可展示所绑定信息源的状态情况。
- FR-TASK-3：可触发分析，支持「全量分析」与「增量分析」两种模式。
- FR-TASK-4：增量分析基于信息源新增内容，仅分析水位线之后的新条目。
- FR-TASK-5：分析调用大模型接口（OpenAI 兼容）；支持逐条分析（per_item）与汇总分析（aggregate）。
- FR-TASK-6：分析结果持久化，可按任务/运行查看。

### 3.4 任务中心（基础模块）

- FR-TC-1：展示系统任务运行列表（信息源同步、分析任务执行）。
- FR-TC-2：展示每次运行的状态、摘要、错误、开始/结束时间。
- FR-TC-3：可查看每次运行的日志详情；可删除运行记录。

### 3.5 系统配置（基础模块）

- FR-CFG-1：只读展示 `config/app.json`（敏感字段脱敏）。
- FR-CFG-2：展示运行时信息（启动时间、PID、版本、监听地址、数据库/日志/数据目录）。

## 4. 非功能需求

- NFR-1：环境相关信息（IP、端口、用户名、密码、绝对路径）不得硬编码，须可配置（`config/app.json` + 环境变量覆盖）。
- NFR-2：展示时间须为北京时间（存储 UTC）。
- NFR-3：下载文件按 年/月/日 存放于 `data/downloads/`。
- NFR-4：`.gitignore` 包含 `logs/`，不包含 `data/` 目录。
- NFR-5：所有功能须有单元测试与冒烟测试且通过。
- NFR-6：支持 Windows（ps1）与 Linux（sh + systemd）两种部署运维方式。
- NFR-7：网页抓取优先使用集中的 WebFetch 服务。

## 5. 信息源类型说明

| 类型 | 数据来源 | 增量方式 |
|---|---|---|
| 官方网站 | 通过 WebFetch 服务抓取站点 HTML，解析文章链接与正文 | 按 URL 去重，新 URL 视为新增 |
| 本地文件夹 | 扫描指定目录下文件（txt/md/pdf/docx/html），抽取文本 | 按文件路径去重，内容变更则更新 |
| FreshRSS | 通过 Google Reader API 拉取指定信息流条目 | 按条目 ID 去重，可选抓取后标记已读 |
